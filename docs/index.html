Filesystem-first template runtime

Reusable analysis templates, with transparent runs.

Linkar sits between ad hoc scripts and heavyweight workflow systems. It gives you reusable templates, portable run artifacts, project-aware chaining, and metadata that humans and agents can inspect.

Why Linkar

Keep the execution model small.

Templates stay standalone

Each template is just a normal directory with a small YAML interface and a runnable entrypoint. Template authors can still test their source scripts directly.

Projects stay readable

project.yaml records pack choices and run history without becoming a hidden workflow engine or database.

Artifacts stay inspectable

Every run lives in a normal directory with results/, .linkar/meta.json, and runtime details that can be inspected later.

Quickstart

The shortest useful path.

1. Install and test

pixi run test
pixi run cli-help

2. Start a project

linkar project init --name demo
cd demo

3. Add a pack

linkar config pack add ~/github/izkf_genomics_pack
linkar templates

4. Run a template

linkar run fastqc --input sample.fastq.gz
linkar project runs
linkar inspect run fastqc_001

Mental Model

Three layers, one direction.

01

Template

Atomic reusable unit with a declared interface and a runnable entrypoint.

02

Pack

Purposeful collection of templates and optional binding defaults.

03

Project

Local run index that tracks outputs, provenance, and active pack choices.
